\chapter{Conclusion and Future Work}
\label{sec:conclusion}

\section{Summary and Main Contributions}

This thesis investigated the Bilevel Parallel Machine Scheduling problem with Adversarial Job Selection, where a leader selects job types within a budget to maximize makespan, anticipating that a follower will optimally schedule jobs to minimize completion time. The problem combines two classic optimization challenges—knapsack selection and parallel machine scheduling—within a hierarchical, adversarial framework.

The main contributions are:
\begin{itemize}
    \item Formal mathematical model for the bilevel knapsack-scheduling problem, including leader and follower objectives and constraints.
    \item Development of a branch-and-bound algorithm with efficient pruning, utilizing both ceiling knapsack and MaxLPT bounds, and incorporating an early optimality detection mechanism that can solve certain instances directly at the root node.
    \item Implementation of dynamic programming for bound computation and incremental LPT scheduling for fast makespan evaluation.
    \item Integration of Gurobi~\cite{gurobi} for solving integer programming subproblems, ensuring optimality in the follower's schedule.
    \item Comprehensive computational study demonstrating practical scalability, pruning effectiveness, and comparison to baseline enumeration.
\end{itemize}

\section{Key Results and Insights}

\begin{itemize}
    \item The branch-and-bound algorithm achieves substantial speedups over complete enumeration, successfully solving instances with up to 26 job types and 12 machines, and demonstrating scalability beyond these sizes in certain cases.
    \item Tight bounds from knapsack scheduling relaxation and LPT rule enable effective pruning, reducing the search space and improving efficiency.
    \item The modular implementation supports efficient node expansion, incremental updates, and memory savings compared to breadth-first approaches.
    \item Gurobi integration ensures that the follower's scheduling problem is solved optimally, validating the correctness of the approach.
    \item Computational experiments confirm the algorithm's robustness across diverse instance types, including uniform, high variance, and correlated job structures.
\end{itemize}

\section{Limitations}

\begin{itemize}
    \item The LPT rule provides a $\frac{4}{3}$-approximation for makespan bounds, which may slightly overestimate contributions from decided items.
    \item There is currently no theory guaranteeing for which instances the optimal solution can be found at the root node, so worst-case complexity remains exponential.
    \item Scalability is limited for very large instances (e.g., 12+ job types or extreme budget/machine counts) due to exponential complexity if the solution cannot be obtained at the root node.
    \item The current model focuses solely on makespan; real-world applications may require multi-objective optimization (cost, energy, risk distribution).
\end{itemize}

\section{Future Directions}

\begin{itemize}
    \item Algorithmic improvements: Enhanced pruning, symmetry breaking, learning-based node selection, and parallel search.
    \item Theoretical analysis: Characterization of instance properties that allow optimal solutions at the root node.
    \item Problem extensions: Unrelated machines, precedence constraints, stochastic processing times, and multi-objective formulations.
    \item Application areas: Production scheduling, capacity planning, project management, and resource allocation under uncertainty.
\end{itemize}


\section{Closing Remarks}

Bilevel optimization is a powerful framework for hierarchical decision-making under uncertainty. This thesis demonstrates that combining principled search, tight bounding, and efficient pruning enables practical solution of complex adversarial scheduling problems. The branch-and-bound approach, supported by dynamic programming and integer programming solvers, provides a solid foundation for future research and real-world deployment.

The methodology is viable for deployment in operational systems, as confirmed by substantial speedups and scalability in computational experiments. Future work should focus on scaling to even larger instances through parallelization and exploring richer problem variants that capture additional real-world complexities. The foundation established here provides a solid platform for such extensions.
